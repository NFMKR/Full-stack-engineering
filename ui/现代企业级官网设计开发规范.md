# 现代企业级官网设计开发规范 (`PROJECT_SPEC.md`)

**[指令]**：你是本项目的前端技术负责人。在生成任何代码之前，必须阅读并严格遵守以下规范。违背此规范的代码将被视为“无效代码”。

-----

## 1\. 核心原则 (Core Principles)

  * **设计风格**：[此处填入，例如：深色科技、工业精密、B2B 专业感]
  * **语义化优先**：严禁全篇 `<div>`。必须使用 HTML5 语义标签 (`<header>`, `<nav>`, `<section>`, `<article>`, `<footer>`)。
  * **移动优先 (Mobile First)**：CSS 编写应优先考虑移动端，通过 `@media (min-width: 768px)` 适配桌面端。
  * **无魔法数值 (No Magic Numbers)**：所有颜色、间距、字号、圆角必须使用 CSS 变量。

-----

## 2\. 基础设计令牌 (Design Tokens)

### A. 色彩系统 (Color Palette)

*必须在 `:root` 中定义，禁止在组件中硬编码 HEX 值。*

```css
:root {
  /* --- 基础背景 (Backgrounds) --- */
  --bg-page:       #0A0A0A; /* 页面主背景 */
  --bg-surface:    #121212; /* 卡片/模块背景 */
  --bg-elevated:   #1E1E1E; /* 悬浮层/弹窗 */

  /* --- 品牌色 (Brand) --- */
  --brand-primary: #0066FF; /* 主行动点 (CTA) */
  --brand-hover:   #0052CC; /* 交互态 */
  --brand-glow:    rgba(0, 102, 255, 0.4); /* 发光效果 */

  /* --- 文字系统 (Typography) --- */
  --text-primary:   rgba(255, 255, 255, 0.95); /* 标题 */
  --text-secondary: rgba(255, 255, 255, 0.75); /* 正文 */
  --text-tertiary:  rgba(255, 255, 255, 0.45); /* 辅助信息 */

  /* --- 功能色 (Functional) --- */
  --border-subtle:  rgba(255, 255, 255, 0.1);
  --border-focus:   var(--brand-primary);
}
```

### B. 排版系统 (Typography)

*使用流体排版 (Fluid Typography) 适配不同屏幕。*

  * **Font Family**: `Inter`, `PingFang SC`, `Microsoft YaHei`, sans-serif.
  * **Scale**:
      * `--font-d1`: `clamp(3rem, 5vw, 4.5rem)` (大标题)
      * `--font-h1`: `clamp(2rem, 3vw, 3rem)` (板块标题)
      * `--font-h2`: `clamp(1.5rem, 2vw, 2rem)` (副标题)
      * \--font-body: `1rem` (正文 16px)
      * \--font-sm:   `0.875rem` (辅助 14px)
  * **Line Height**: 标题 `1.2`，正文 `1.6`。

### C. 空间与间距 (Spacing & Layout)

*使用 4px 网格系统的倍数。*

  * **间距变量**:
      * `--space-xs`: `4px`
      * `--space-sm`: `8px`
      * `--space-md`: `16px`
      * `--space-lg`: `32px` (组件内间距)
      * `--space-xl`: `64px` (模块间距 - 移动端)
      * `--space-2xl`: `120px` (模块间距 - 桌面端)
  * **容器宽度**:
      * `--container-width`: `1280px`
      * `--container-padding`: `20px`

-----

## 3\. 组件原子规范 (Atomic Components)

### A. 按钮 (Buttons)

  * **规范**：按钮必须有明确的 Hover、Active 状态和 Focus 轮廓（无障碍访问）。
  * **代码范式**：
    ```css
    .btn {
      display: inline-flex;
      align-items: center;
      justify-content: center;
      padding: 12px 24px;
      font-weight: 600;
      border-radius: 4px; /* 工业风通常圆角较小 */
      transition: all 0.3s ease;
    }
    .btn-primary {
      background: var(--brand-primary);
      color: white;
    }
    .btn-primary:hover {
      background: var(--brand-hover);
      box-shadow: 0 0 15px var(--brand-glow);
      transform: translateY(-2px);
    }
    ```

### B. 卡片 (Cards)

  * **规范**：企业官网的卡片（如“服务项目”、“案例”）应具备层级感。
  * **风格**：微边框 + 极其微弱的背景色。
  * **代码范式**：
    ```css
    .card {
      background: var(--bg-surface);
      border: 1px solid var(--border-subtle);
      padding: var(--space-lg);
      border-radius: 8px;
    }
    .card:hover {
      border-color: var(--border-focus); /* Hover 时边框高亮 */
    }
    ```

### C. 导航栏 (Navbar)

  * **规范**：滚动时固定顶部，背景需做“玻璃拟态 (Glassmorphism)”处理。
      * `backdrop-filter: blur(10px);`
      * `background: rgba(10, 10, 10, 0.8);`

-----

## 4\. 布局结构规范 (Layout Patterns)

所有页面板块（Section）必须遵循以下 HTML 结构：

```html
<section class="section-wrapper" id="features">
  <div class="container">
    <div class="section-header">
      <span class="tag">核心技术</span>
      <h2 class="title">AI 驱动的工业引擎</h2>
      <p class="desc">利用大模型重构生产流程...</p>
    </div>
    
    <div class="grid-content">
      </div>
  </div>
</section>
```

  * **CSS Grid**: 推荐使用 Grid 进行二维布局（如 3 列卡片）。
      * `display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: var(--space-lg);`

-----

## 5\. 交互与动效 (Interaction & Motion)

  * **滚动库**：推荐使用 **Lenis** 实现平滑滚动。
  * **入场动画**：
      * 所有大标题和图片进入视口时，必须有 `Fade Up` 效果。
      * **参数**：`Y: 30px -> 0`, `Opacity: 0 -> 1`, `Duration: 0.8s`, `Ease: Power2.out`。
  * **微交互**：
      * 链接/按钮 Hover 时，透明度不能低于 0.8。
      * 图片 Hover 时，建议轻微放大 (`scale: 1.05`)。

-----

## 6\. 图片与多媒体 (Assets)

  * **图片格式**：必须使用 WebP。
  * **Lazy Load**：首屏以外的图片必须添加 `loading="lazy"`。
  * **Alt 属性**：所有 `<img>` 标签必须包含对 SEO 友好的 `alt` 描述。

-----

## 7\. 开发负面清单 (Negative Constraints)

  * **禁止**使用 `float` 布局。
  * **禁止**使用 `!important` 覆盖样式（除非处理第三方库）。
  * **禁止**把样式写在 JS 里（除动态计算的值外）。
  * **禁止**深层嵌套 CSS（超过 3 层需重构类名）。
