# [Project: Golden Signal] UI 开发设计规范指南

**[指令]**：你是本项目的前端设计架构师。请严格遵循以下“黑金赛博”风格进行开发。所有 UI 元素必须体现“从黑暗噪音中提炼价值”的隐喻。

## 1\. 核心设计哲学 (Design Philosophy)

  * **核心隐喻**：**“深渊矿工 (The Abyss Miner)”**。
      * **黑暗 (The Noise)**：社交媒体的海量废话、冗余信息。表现为深邃的黑、噪点纹理、模糊的背景。
      * **金子 (The Signal)**：高价值的需求、洞察、数据。表现为发光的金、锐利的边缘、高亮的流光。
  * **视觉风格**：Cyberpunk Luxury (赛博奢华) + Precision Data (精密数据)。
  * **用户感受**：像在驾驶一艘深空飞船，仪表盘只显示最关键的导航金线。

## 2\. 视觉规范 (Visual Specs)

### A. 色彩体系 (Color Palette)

*不使用平面的黄，金子必须是“流动”的；不使用死板的黑，黑暗必须是“深邃”的。*

```css
:root {
  /* --- 1. 深渊基底 (The Void) --- */
  --bg-void:       #000000; /* 极致纯黑 (用于大背景) */
  --bg-depth:      #050505; /* 次级背景 (叠加噪点) */
  --bg-surface:    #0F1116; /* 卡片/模块背景 (极深蓝灰) */
  --bg-glass:      rgba(15, 23, 42, 0.6); /* 玻璃态背景 */

  /* --- 2. 流光金 (The Gold) --- */
  --gold-primary:  #FFD700; /* 核心金 (纯金) */
  --gold-dim:      #C5A000; /* 暗金 (未激活状态) */
  --gold-light:    #FFF2B2; /* 亮金 (高光) */
  /* 流光渐变：核心视觉资产 */
  --gold-gradient: linear-gradient(135deg, #FFD700 0%, #FDB931 50%, #9C7C1C 100%);
  --gold-shimmer:  linear-gradient(90deg, transparent, rgba(255, 215, 0, 0.5), transparent);

  /* --- 3. 信号与噪音 (Text) --- */
  --text-signal:   #FFFFFF; /* 核心信息 (纯白) */
  --text-noise:    #64748B; /* 噪音/辅助信息 (蓝灰，融入背景) */
  --text-gold:     #FFD700; /* 强调数据/关键洞察 */

  /* --- 4. 边框与光效 (Borders & Glows) --- */
  --border-dim:    rgba(255, 255, 255, 0.1);
  --border-gold:   rgba(255, 215, 0, 0.5);
  --glow-gold:     0 0 20px rgba(255, 215, 0, 0.3); /* 霓虹光晕 */
}
```

### B. 排版与字体 (Typography)

  * **字体家族**：
      * **UI/正文**：`Inter` 或 `Space Grotesk` (带一点科技感的无衬线)。
      * **数据/代码**：`JetBrains Mono` 或 `Fira Code` (等宽字体，强调数据的精密感)。
  * **排版策略**：
      * **Data First**：关键数字使用 `JetBrains Mono`，字重 `700`，颜色 `--text-gold`。
      * **Label Dim**：标签和说明文字使用 `--text-noise`，字号极小 (`12px`)，大写 (`uppercase`)，增加字母间距 (`letter-spacing: 0.05em`)。

### C. 材质与纹理 (Texture & Effects)

  * **背景噪点 (The Noise)**：在 `--bg-void` 上覆盖一层透明度极低 (3%) 的 SVG 噪点，隐喻“社交媒体的混乱”。
  * **赛博玻璃 (Cyber Glass)**：
      * `backdrop-filter: blur(16px);`
      * `background: linear-gradient(180deg, rgba(255,255,255,0.03) 0%, rgba(255,255,255,0.01) 100%);`
      * `border: 1px solid rgba(255, 215, 0, 0.15);` (微弱的金边)

## 3\. 组件设计规范 (Atomic Design)

### A. 按钮：炼金术 (The Alchemy Button)

  * **主按钮 (Primary)**：不使用纯色块，使用“金边镂空”或“暗金填充”。
      * **常态**：深色背景，1px 金色边框，文字金色发光。
      * **Hover**：背景变为 `--gold-gradient`，文字变黑，产生强烈的“激活”感。
      * **光效**：Hover 时添加 `box-shadow: var(--glow-gold)`。

<!-- end list -->

```css
.btn-gold {
  background: transparent;
  border: 1px solid var(--gold-primary);
  color: var(--gold-primary);
  text-shadow: 0 0 8px rgba(255, 215, 0, 0.5);
  transition: all 0.3s ease;
}
.btn-gold:hover {
  background: var(--gold-gradient);
  color: #000;
  text-shadow: none;
  box-shadow: var(--glow-gold);
}
```

### B. 卡片：数据全息板 (Holographic HUD)

  * **隐喻**：悬浮在黑暗中的全息屏幕。
  * **样式**：
      * 直角或极小圆角 (`4px`)，强调工业精密感。
      * **四角装饰**：在卡片的四个角落添加 10px 长的金色直角线段（HUD 风格）。
  * **高亮逻辑**：当卡片内包含“高价值需求”时，卡片边缘呼吸闪烁金色光芒。

### C. 数据可视化 (Charts)

  * **背景**：透明或深黑。
  * **线条**：极细 (`1px`)，颜色使用 `--gold-primary`。
  * **区域填充**：使用线性渐变 `rgba(255, 215, 0, 0.2)` 到 `transparent`。
  * **网格线**：虚线，颜色 `--border-dim` (深灰)，隐喻“坐标系”。

## 4\. 动效编排 (Motion)

### 4.1. 扫描 (Scan)

* **隐喻**：页面加载时，一道金色的光线（1px height）从屏幕顶部快速扫描到底部，随后内容浮现。隐喻“正在扫描噪音”。
* **实现**：使用一个宽度为 $100\%$, 高度为 $1px$ 的元素，应用 `var(--gold-shimmer)` 渐变，并使用 CSS 动画 `translateY` 快速从 `-100vh` 移动到 `100vh`。

### 4.2. 流光 (Shimmer)

* **隐喻**：关键的数据卡片边缘，每隔 5 秒有一道流光划过边框。
* **实现**：使用伪元素 (`::before` 或 `::after`) 在边框上创建一段窄长的 `var(--gold-shimmer)` 渐变，并使用 `keyframe` 动画使其沿边框移动。用于强调 **“实时数据流”**。

### 4.3. 故障风 (Glitch - 可选)

* **隐喻**：当显示“垃圾信息/噪音”被过滤掉时，使用轻微的 Glitch 效果消失。
* **实现**：用于**数据过滤/清理**完成时，被清除的元素（例如低价值的评论）短暂出现颜色通道分离和位移效果后，快速 `opacity: 0` 消失。

## 4.5. 🚀 仪表板微动效 (Dashboard Micro-Interactions) **\[新增\]**

本节动效旨在提供“深空飞船”仪表盘的精密反馈，确保操作的每一个环节都有清晰的“机械”响应。

### A. 导航栏 (Navigation)

* **常态**：导航项使用 `--text-noise`。
* **激活**：选中的导航项文字颜色切换为 `--text-gold`，并在文字下方添加一个 **2px 宽、10px 高** 的金色**垂直光标条**，该光标条应伴随轻微的垂直呼吸动效。

### B. 输入框与选择器 (Inputs & Selectors)

* **焦点状态 (Focus)**：当输入框获得焦点时，边框颜色从 `--border-dim` 切换为 `--border-gold`。同时，在输入框的底部和右侧添加一个**“精密定位线”**的动画：两条 1px 宽的金色线条，从输入框的中心向边缘快速延伸，并在角落处停留 0.1 秒后消失。

### C. 数据点提示 (Tooltip / Hover Data)

* **图表交互**：当鼠标悬停在图表上的数据点 (`--gold-primary` 圆点) 上时，该数据点应瞬间放大 ($scale(1.5)$) 并产生一圈**极细的金色环形脉冲**（单次）。
* **提示框**：显示的 Tooltip 背景必须使用 `--bg-glass` 材质（带有磨砂玻璃效果），以强调它是**实时生成**的全息信息。

### D. 数据更新 (Data Refresh)

* **更新效果**：当新的“黄金信号”数据载入并替换旧数据时，不应直接跳变。
    * **旧数字**：`opacity: 0` 渐隐。
    * **新数字**：使用 **数字滚动动画** (Counter Animation) 从旧值滚动到新值，并在滚动过程中文字短暂使用 `--gold-light`，滚动结束后恢复为 `--text-gold`，强调数据的**“发现过程”**。

### E. 模块激活 (Card Activation)

* **点击 / 选中**：卡片被点击或选中时，除了边框的呼吸闪烁，卡片本身应伴随轻微的 $translateZ(5px)$ **上浮**效果，随后恢复原位，隐喻“数据模块被成功调取”。

总结：如何避免“土豪金”？

要实现高级的 Cyberpunk 质感，关键在于**克制**：

1.  **黑要够黑**：$90\%$ 的面积是黑色/深灰色。
2.  **金要够亮**：金只用于**最有价值**的数据（数字、按钮、图表线）。
3.  **光要微弱**：所有的发光效果（Glow）必须是扩散的、柔和的，不要生硬的描边。
