## 1\. 整体样式风格 (Global Style & Vibe)

**核心定调**：

  * **风格关键词**：[明亮、亲和、高效、多彩]
  * **视觉参照**：Smallpdf.com
  * **体验目标**：用户进入即用，无心理负担。大色块区分功能，高对比度文字，圆润的交互触感。

### A. 视觉规范 (Visual Specs)

**色彩体系 (CSS Variables)**
*不要使用死板的纯黑纯白，使用高明度色彩营造“工具的愉悦感”。*

```css
:root {
  /* --- 品牌色 (Brand) --- */
  /* Smallpdf风格通常每个工具一种颜色，这里定义主品牌色 */
  --brand-primary: #FF3C41; /* 活力红 (主行动点/Logo) */
  --brand-hover:   #D92D32;
  
  /* --- 背景体系 (Backgrounds) --- */
  --bg-page:       #F7F9FC; /* 极淡的蓝灰色背景，保护视力 */
  --bg-surface:    #FFFFFF; /* 卡片/弹窗背景 (纯白) */
  --bg-input:      #F0F3F7; /* 输入框浅灰背景 */

  /* --- 文字体系 (Text) --- */
  --text-main:     #171717; /* 主要文字 (近黑) */
  --text-body:     #474747; /* 正文文字 (深灰) */
  --text-dim:      #9CA3AF; /* 辅助/占位符 */

  /* --- 辅助色 (Utility) --- */
  --color-success: #2ECC71;
  --color-error:   #E74C3C;
  --ad-placeholder:#E5E7EB; /* 广告位灰色 */

  /* --- 质感 (Effects) --- */
  --shadow-card:   0 4px 12px rgba(0, 0, 0, 0.05); /* 柔和投影 */
  --shadow-hover:  0 8px 24px rgba(0, 0, 0, 0.12); /* 悬浮投影 */
  --radius-sm:     4px;
  --radius-md:     8px;  /* 按钮/输入框 */
  --radius-lg:     12px; /* 卡片 */
  --radius-xl:     24px; /* 弹窗 */
}
```

### B. 布局与响应式 (Layout)

  * **基准断点**：`Desktop (1440px)`, `Tablet (768px)`, `Mobile (375px)`.
  * **最大容器**：`max-width: 1200px; padding-x: 24px; margin: 0 auto;`
  * **移动端特殊处理**：
      * **汉堡菜单**：\< 768px 时，导航栏收纳为 Hamburger Menu。
      * **卡片堆叠**：工具网格从 `Grid-4` 变为 `Grid-1`。
      * **广告位调整**：侧边栏广告在移动端自动转为“底部粘性 (Sticky Bottom)”或隐藏。

## 2\. 商业化与合规元素 (Commercial & Legal)

### A. 广告位预留 (Ad Slots)

*必须符合 Google AdSense / IAB 标准尺寸，且包含防抖动 (CLS) 处理。*

1.  **首页/列表页**:
      * **Top Banner**: `728x90` (Desktop) / `320x50` (Mobile).
      * 位置：导航栏下方，Hero 区域上方。
2.  **工具详情页**:
      * **Sidebar Rect**: `300x250` (Desktop). 右侧固定。
3.  **开发占位代码**:
    ```html
    <div class="ad-slot" style="background: var(--ad-placeholder); display: flex; align-items: center; justify-content: center; color: var(--text-dim); font-size: 12px;">
       <span class="label">Ad Space</span>
       </div>
    ```

### B. 合规性页脚 (Footer Legal)

*Google Ads 审核必须项。*

  * **布局**：四列布局（产品、公司、支持、法律）。
  * **必须包含链接**：
      * Privacy Policy (隐私政策)
      * Terms of Use (使用条款)
      * Contact Us (联系我们) - *需包含真实邮箱链接*
      * Cookie Policy
  * **样式**：深色底 (`#171717`) 或 纯白底，小字号 (`14px`)。

## 3\. 具体业务组件规范 (Core Components)

### A. 导航栏 (Navbar)

  * **高度**：`64px`，`sticky top-0`，背景色 `var(--bg-surface)`，下边框 `1px solid var(--bg-input)`。
  * **左侧**：Logo (SVG 图标 + 粗体文字 "Simple**Tools**")。
  * **中间**：Dropdown 菜单 ("所有工具", "压缩", "转换")。
  * **右侧**：
      * **Log in (登录)**：透明背景，Hover 变灰。
      * **Free Trial (免费试用)**：实心品牌色按钮 (`bg-brand-primary`)。

### B. 首页基础布局 (Homepage)

1.  **Hero Section**:
      * 居中对齐。
      * **H1**: `font-size: 48px; font-weight: 800;` (例如: "We make PDF easy")。
      * **Sub**: `font-size: 20px; color: var(--text-body);`。
2.  **Tool Grid (核心区域)**:
      * **Layout**: `display: grid; grid-template-columns: repeat(auto-fill, minmax(260px, 1fr)); gap: 24px;`
      * **Card Style**:
          * `bg: white`, `padding: 24px`.
          * **Icon**: 48x48px SVG 图标，每种工具类别不同颜色（PDF=红, IMG=黄, Convert=绿）。
          * **Title**: `font-weight: 700; font-size: 18px;`.
          * **Desc**: `font-size: 14px; color: var(--text-body);`.
          * **Interaction**: Hover 时 `transform: translateY(-5px); box-shadow: var(--shadow-hover);`。

### C. 登录/注册弹框 (Auth Modal)

*仿 Smallpdf 的弹窗体验，不跳转新页面。*

  * **触发**：点击导航栏 "Log in"。
  * **Backdrop**: `background: rgba(0,0,0,0.5); backdrop-filter: blur(4px);`
  * **Modal Box**:
      * `width: 400px; background: white; border-radius: var(--radius-xl); padding: 40px;`
      * **Header**: "Welcome back" (H2).
      * **Social Buttons**: "Continue with Google" (大按钮, 带 Google Logo, 灰色描边)。
      * **Divider**: "OR" 线条。
      * **Form**: Email Input + Password Input (大圆角, `height: 48px`).
      * **Footer**: "No account? Create one."
  * **动效**: `Scale: 0.9 -> 1`, `Opacity: 0 -> 1` (Spring 动画)。

### D. 订阅页固定规范 (Pricing Page)

*标准 SaaS 商业化页面。*

  * **Layout**: `Grid-3` (Free / **Pro** / Team).
  * **Card Common**: `border: 1px solid var(--bg-input); border-radius: var(--radius-lg); padding: 32px;`
  * **Pro Card (重点推荐)**:
      * `border: 2px solid var(--brand-primary);`
      * **Badge**: 顶部绝对定位标签 "Most Popular" (`bg-brand-primary`, text white).
      * **Button**: 全宽品牌色按钮。
  * **Typography**:
      * Price: `font-size: 48px; font-weight: 800;`
      * Currency: `font-size: 24px; vertical-align: super;`
      * Features: 列表项使用 ✅ (绿色 SVG 对勾)，禁止换行。

## 4\. 功能模块预留 (Feature Module Slot)

*此部分为动态功能区。当需要开发具体工具（如“PDF 转 Word”或“图片压缩”）时，请使用以下模板填充并执行。*

```markdown
### [动态模块] 具体工具功能定义
**当前任务**：[待填充，例如：开发 PDF 压缩功能的前端交互]

**UI 布局要求**：
1.  **Dropzone (上传区)**：
    * 占据屏幕 60% 高度 (Hero 区域)。
    * 样式：虚线边框，背景色 `var(--bg-page)`，中心是大图标 + "Choose File" 按钮。
    * 交互：支持拖拽 (Drag & Drop)，Hover 时边框变色。
2.  **Processing State (处理中)**：
    * 隐藏 Upload 区，显示进度条 (Progress Bar)。
    * 文案："Compressing your file..."
3.  **Result State (结果页)**：
    * 左侧：文件略缩图 + 压缩率数据 ("Saved 40%!")。
    * 右侧：大号 "Download" 按钮 + "Save to Cloud" 选项。
    * 底部/侧边：展示 [广告位]。

**逻辑要求**：
* 限制：未登录用户每天限 2 次，超过弹出 [Auth Modal]。
* API：预留 `POST /api/process` 接口调用位置。
```
